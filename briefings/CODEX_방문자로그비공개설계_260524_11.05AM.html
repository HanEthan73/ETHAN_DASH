CODEX 방문자 로그 비공개 설계

먼저 결론입니다.
GitHub Pages만으로는 “누가 왔는지”를 정확히 알 수 없습니다. GitHub Pages는 정적 호스팅이라 서버 접속 로그를 대표님에게 제공하지 않습니다. 현재 페이지의 방문자수 폴백은 브라우저 로컬 기준이라 전체 방문자 분석용이 아닙니다. “나만 보고 싶은 날짜별 방문 로그”를 만들려면 별도 로그 저장소가 필요합니다.

1. 가능한 방식 비교

방식누가 왔는지날짜 로그나만 보기장점단점추천
GitHub Pages 기본불가불가-추가 설정 없음방문자 IP/기기/시간 로그 접근 불가불가
GoatCounter / Plausible개인 식별 불가가능가능간단하고 개인정보 부담 낮음정확한 개인 이름은 모름추천
Cloudflare Web Analytics개인 식별 불가가능가능무료, 속도 영향 낮음GitHub Pages 도메인 앞에 Cloudflare 설정 필요추천
Google Analytics개인 식별 불가가능가능기능 강함설정 복잡, 개인정보/쿠키 이슈보류
직접 로그 APIIP/UA 정도 가능가능가능대표님 원하는 형태로 저장 가능서버/DB/보안 필요. 공개 API 키 노출 위험고급

2. “나만 보기” 현실적 구조

구성설명실행 방법
공개 브리핑 페이지현재 GitHub Pages 유지. 방문자에게 별도 관리자 정보 노출 없음.index.html에 분석 스크립트 1줄 삽입.
비공개 분석 대시보드GoatCounter/Plausible/Cloudflare 계정에 로그인한 대표님만 확인.대시보드 URL은 공개 페이지에 노출하지 않음.
날짜별 로그일자, 페이지, 국가/지역, referrer, 기기 종류 정도 확인.분석 도구가 자동 수집.
개인 식별이름까지는 불가. IP도 개인정보라 저장/표시를 조심해야 함.“누가”보다 “언제 어떤 페이지를 얼마나 봤는지” 중심으로 보는 게 안전.

3. 추천 실행안

순서작업담당결과
1GoatCounter 무료 계정 생성 또는 Cloudflare Analytics 연결대표님 계정 필요비공개 관리자 대시보드 생성
2추적 스크립트 1줄을 briefings/index.html에 삽입Codex 가능방문 로그 수집 시작
3관리자 링크는 로컬 메모/이메일에만 보관대표님방문자에게는 안 보임
4매일 방문 요약을 Gmail 명령으로 요청Codex 가능날짜별 방문 추세 보고 가능

4. 최종 답변

  • 지금 상태: 방문자수는 외부 API 실패 시 로컬 브라우저 기준으로만 표시됩니다. 전체 방문자 로그가 아닙니다.
  • 누가 왔는지: GitHub Pages만으로는 확인 불가입니다.
  • 날짜 로그: GoatCounter, Plausible, Cloudflare Analytics 같은 별도 도구가 필요합니다.
  • 나만 보기: 분석 도구 대시보드를 비공개로 두면 가능합니다.
  • 추천: 개인정보 부담이 낮은 GoatCounter 또는 Cloudflare Web Analytics부터 시작하는 것이 맞습니다.

주의: 방문자 IP, 기기정보, 위치정보는 개인정보에 해당할 수 있으므로 공개 페이지에 직접 노출하면 안 됩니다.
