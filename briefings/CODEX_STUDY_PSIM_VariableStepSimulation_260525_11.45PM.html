1. 먼저 설명: Fixed Step과 Variable Step의 차이

구분Fixed StepVariable Step대표님 회로에서의 의미
계산 방식처음부터 끝까지 같은 시간 간격으로 계산변화가 빠른 곳은 작게, 느린 곳은 크게 계산60Hz 입력 한 주기를 오래 돌릴 때 유동 step이 속도 이점이 있음

단점고주파 스위칭 기준으로 잡으면 60Hz 전체가 너무 느림Max Step이 크면 edge, peak, zero-crossing을 놓칠 수 있음CLLC SR timing 검증에는 variable만 믿으면 위험

4. 실전 설정 순서

순서조치판단 기준
1평균모델로 60Hz 전체 동작 먼저 확인Vbus, 입력전류 envelope, 제어 루프가 정상인지 빠르게 본다
2스위칭 모델에서 Variable Step + Max Step 제한Max Step은 최소 Tsw/50, SR/ZCD는 더 보수적으로
3Print Step은 크게 두고 필요한 노드만 저장파일 용량과 표시 속도를 줄인다
4문제 구간 2~3ms만 잘라 Fixed Step 재검증Variable 결과와 peak, zero-crossing, ripple이 같은지 비교
5결과가 1~2% 이내면 Variable Step을 작업용으로 채택손실/SR timing은 계속 fixed 검증을 붙인다

5. 최종 요약

질문답
고정 step 안 하고 유동 step으로 하면 되나?60Hz 전체 파형/초기 탐색은 된다.
그럼 항상 유동 step만 쓰면 되나?아니다. SR, deadtime, switching loss는 fixed step 또는 아주 작은 Max Step 검증이 필요하다.
가장 중요한 설정은?Max Step. Variable Step의 자유도를 주되, 스위칭 이벤트를 건너뛰지 못하게 막는 안전장치다.
대표님에게 맞는 결론PFC 60Hz는 variable로 빠르게 돌리고, CLLC SR 매핑은 문제 구간만 fixed로 확대 검증한다.

6. CODEX 인사이트

시뮬레이션 속도를 올리는 핵심은 “step을 크게 한다”가 아니라 “관심 없는 시간에는 크게, 물리 이벤트가 있는 순간에는 작게”입니다. 대표님 회로에서는 60Hz envelope와 수십~수백 kHz switching이 한 파일 안에 섞여 있으므로, 하나의 고정 step으로 전부 해결하려는 순간 시간이 폭발합니다. 반대로 variable step을 방치하면 가장 중요한 SR zero-crossing을 지나쳐 버립니다. 따라서 좋은 전략은 solver를 믿는 것이 아니라 solver에게 어디까지 건너뛰어도 되는지를 숫자로 명령하는 것입니다.
