🖥️ CPU vs GPU vs HBM — 역할·구조·장단점 완전 분석

① 약자 먼저

CPU = Central Processing Unit (중앙 처리 장치)
컴퓨터의 두뇌. 모든 연산·명령 처리의 중심.
GPU = Graphics Processing Unit (그래픽 처리 장치)
원래 화면 렌더링용 → AI·과학 계산으로 확장.
HBM = High Bandwidth Memory (고대역폭 메모리)
GPU 옆에 적층해서 붙이는 초고속 메모리. AI 연산의 병목 해결사.
ASIC = Application-Specific Integrated Circuit (주문형 반도체)
특정 용도만을 위해 맞춤 설계된 칩. 브로드컴 TPU 계열.
ALU = Arithmetic Logic Unit (산술 논리 장치)
실제 덧셈·뺄셈·비교 등 연산을 수행하는 회로 블록.
FLOPS = Floating Point Operations Per Second
초당 부동소수점 연산 수. 컴퓨팅 성능 단위.
DRAM = Dynamic Random Access Memory
일반 메모리. HBM은 DRAM을 수직으로 여러 층 쌓은 것.

② CPU — "만능 해결사"

역할: 컴퓨터의 중앙 두뇌. 운영체제, 프로그램 실행, 분기·조건문 등 복잡한 순차 논리 처리.

핵심 설계 철학: 하나의 복잡한 작업을 최대한 빠르게 처리 (Latency 최소화)
8~64개
코어 수 (고성능)
4~6GHz
클럭 속도
대용량
캐시 메모리
CPU 내부 구조 (코어 1개 기준):
  • ALU: 실제 연산 수행
  • 제어 유닛(CU): 명령어 해석·분기 예측
  • 캐시(L1/L2/L3): 자주 쓰는 데이터를 가까이 보관
  • 파이프라인: 명령어 동시 처리 (fetch → decode → execute → writeback)
장점단점

③ GPU — "공장 조립 라인"

역할: 동일한 단순 연산을 수천 개 동시에 처리. 원래 픽셀 렌더링용이었으나 행렬 연산 구조가 AI 딥러닝과 완벽히 일치하여 AI 가속기로 전용.

핵심 설계 철학: 많은 작업을 동시에 처리 (Throughput 최대화)
16,384개
CUDA 코어 수 (H100)
~1.8GHz
클럭 속도 (CPU보다 낮음)
3.35TB/s
메모리 대역폭 (H100)
GPU가 AI에 최적인 이유 — 행렬 곱셈:
AI의 핵심 연산 = 행렬 곱셈 (Matrix Multiplication)
$$\mathbf{Y} = \mathbf{W} \cdot \mathbf{X} + \mathbf{b}$$
\(\mathbf{W}\): 가중치 행렬, \(\mathbf{X}\): 입력, \(\mathbf{b}\): 편향
→ 수백만 개의 곱셈·덧셈을 동시 수행 → GPU 수천 코어가 병렬 처리
예를 들어 \(1024 \times 1024\) 행렬 곱셈 = 약 \(10^9\) 번 연산 → CPU는 순차적으로, GPU는 동시에.
장점단점
✅ 병렬 연산 압도적
✅ AI/딥러닝 행렬 연산 최적화
✅ 대역폭 매우 높음
✅ CUDA 생태계 (NVDA)		 ❌ 단일 복잡 연산은 CPU보다 느림
❌ 분기·조건문 비효율
❌ 전력 소모 큼 (H100 = 700W)
❌ 단독으로 작동 불가 (CPU 보조 필요)

④ HBM — "GPU 옆에 붙은 초고속 창고"

왜 필요한가 — 메모리 병목 문제:

GPU 코어가 아무리 빨라도 데이터를 메모리에서 가져오는 속도가 느리면 코어가 놀게 됩니다.

일반 GDDR 메모리: 대역폭 ~500GB/s
HBM3e (H100): 대역폭 3.35TB/s → 약 6~7배 빠름
HBM 구조 — 3D 적층:

일반 DRAM을 수직으로 8~12층 쌓고, GPU 다이 옆에 패키지 레벨로 붙임 (인터포저 연결).
대역폭 계산 (HBM3e 기준)
$$BW = \text{버스폭} \times \text{클럭} \times 2 = 1024\text{bit} \times 1.638\text{GHz} \times 2 \approx 3.35\text{ TB/s}$$
일반 DDR5: 64bit × 4.8GHz × 2 ≈ 76.8 GB/s → HBM 대비 44배 낮음
메모리 종류대역폭용량위치용도

⑥ AI 데이터센터에서 3개가 협력하는 방식

비유: 공장 생산라인

CPU = 공장장 → 어떤 작업을 할지 지시, 순서 관리
GPU = 조립 라인 작업자 수천 명 → 실제 AI 행렬 연산 수행
HBM = 바로 옆 부품 창고 → 작업자에게 실시간으로 부품(데이터) 공급

창고(HBM)가 느리면 작업자(GPU 코어)가 놀게 됩니다. 그래서 AI 성능 = GPU + HBM 세트가 중요.
